# AI-Native Terminal-First IDE — 要件定義書 (specs.md)

## 1. プロジェクト概要

AIエージェント（Claude Code, OpenCode 等）を複数同時に走らせることを前提とした、**ターミナルが主役のAIネイティブ開発環境**。  
VS Code の組み込みは断念し、**Custom Electron Shell の中で必要な機能を直接実装する**アーキテクチャを採用する。

---

## 2. アーキテクチャ方針

### 2.1 全体構成

```
┌──── Custom Electron Shell (自作) ──────────────────────────────┐
│                                                                  │
│  [Workspace Bar]  │  [Terminal Panes (xterm.js + node-pty)]    │
│  Slack風タブ      │                                             │
│  (完全自作)       │   ─── 右パネル (Right Activity Bar) ───     │
│                   │  📂 Explorer  │ ファイルツリー表示          │
│                   │  📄 PDF       │ PDF ビューア                │
│                   │  🖼️ Image     │ 画像ビューア                │
│                   │  🌐 Browser   │ 組み込みブラウザ            │
└──────────────────────────────────────────────────────────────── ┘
```

### 2.2 技術スタック

| 領域 | 技術 |
|---|---|
| シェル | **Electron** (Node.js + Chromium) |
| ターミナルUI | **xterm.js** (レンダリング) + **node-pty** (PTY制御) |
| ファイルビューア | Electron `<webview>` / HTML Canvas |
| PDF ビューア | `<webview>` に PDF を `file://` で読み込み (Chromium内蔵PDF対応) |
| 画像ビューア | `<img>` タグで直接表示（サポート形式：jpg, png, gif, webp, svg） |
| ブラウザパネル | `<webview>` + URL バー（Playwright に類似したページ遷移） |
| ワークスペースバー | Electron の BrowserWindow 内 HTML/CSS/JS で完全自作 |
| IPC | Electron の `ipcMain` / `ipcRenderer` |

## 3. 機能要件

### 3.1 ワークスペースバー（左側パネル）

- 画面左端に縦並びのワークスペースタブを表示
- 幅が広いときはアイコン＋フォルダ名、狭いときはアイコンのみ（Slack風）
- ドラッグで幅を可変にできる
- クリックで表示中のターミナルペイン群を即時切り替え

### 3.2 ターミナルエリア（メイン領域）

- xterm.js + node-pty によるターミナルを自作でレンダリング
- **マルチペイン表示**: tmux風にグリッド分割で複数エージェントを同時表示可能
- **Broadcast機能**: 全ペインへ同じ入力を一斉送信
- ワークスペースごとにペインのセット・状態を保持・復元

### 3.3 右パネル（Right Activity Bar + 各ビューア）

右アクティビティバーのアイコンをクリックして以下のビューを切り替える：

#### 📂 Explorer（ファイルエクスプローラー）
- ワークスペースのフォルダツリー表示
- ファイルをクリックで対応するビューアに自動ルーティング：
  - `.pdf` → PDFビューア
  - `.jpg/.png/.gif/.webp/.svg` → 画像ビューア
  - その他 → テキストファイルビューア（既存）

#### 📄 PDF ビューア
- Electron 組み込みの PDF 表示機能（`<webview>` + `file:// URL`）を利用
- ファイルパス受け取り → webview の src に直接セット
- ズーム・スクロール対応（Chromium 内蔵の PDF ビューア機能）

#### 🖼️ 画像ビューア
- `<img>` タグで画像を直接表示
- 拡大・縮小（マウスホイール / ピンチ）対応
- ファイル名・解像度・サイズをツールバーに表示

#### 🌐 ブラウザパネル
- `<webview>` で自由な URL を表示（Cursor の Browser View に相当）
- URLバー：テキスト入力 + Enter でナビゲート
- ← / → / 🔄 ナビゲーションボタン
- 開発者ツール呼び出しボタン（`webview.openDevTools()`）
- **Playwright ALT (MCP & Skill) 連携**:
  - ターミナル上のエージェントが利用できる「playwright_ALT」スキルまたは MCP サーバーを提供。
  - **MCP サーバー**: Electron 内で起動し、ブラウザ操作（クリック、入力、スクリプト実行、スクリーンショット）をツールとして公開。
  - **スキル**: エージェントが接続情報を自動取得し、`playwright_ALT.screenshot()` 等の関数を介して内蔵ブラウザと対話可能にする。
  - ターミナルの環境変数（`MCP_PORT`, `BROWSER_CDP_URL` 等）を通じて自動セットアップ。

### 3.4 ワークスペース管理
- ユースケース：
  - localhost で動いている Web アプリのプレビュー
  - AI エージェントによる自動ブラウジング・テスト実行・DOM操作報告
  - API ドキュメント / GitHub を横に置いて参照
  - AI エージェント作成の Web UI を即座に確認

### 3.4 ワークスペース管理

- ワークスペース = ローカルディレクトリ（フォルダパス）
- 設定ファイル（JSON）にワークスペース一覧を永続化
- 新規ワークスペース追加：フォルダ選択ダイアログ
- ワークスペース削除・並び替え

### 3.5 AIエージェント連携

- 各ワークスペースのターミナルで Claude Code / OpenCode を手動 or 自動起動
- コンテキスト共有：開いているファイルのパスをターミナルのエージェントに送信
- 環境変数の自動注入（ワークスペースごとに `.env` 等を設定）

---

## 4. 非機能要件

- macOS 優先（将来的にLinux対応）
- 起動時間：初回以降 3秒以内
- ターミナルの入力遅延：< 10ms
- ブラウザパネルは Electron の `webSecurity` を適切に管理しローカルコンテンツを安全に表示

---

## 5. 開発フェーズ

### Phase 1: Electron シェルの骨格 (MVP) ✅ 完了

1. Electron アプリの雛形作成
2. xterm.js + node-pty でターミナルを表示・操作できる
3. ワークスペースバーの基本UI
4. ターミナルペインのマルチ表示

### Phase 2: 右パネル ビューア群 ← **現在のフェーズ**

5. **PDF ビューア**：`<webview>` で `file://` URL を介してPDFをネイティブ表示
6. **画像ビューア**：`<img>` + ズーム機能
7. **ブラウザパネル**：URLバー付き `<webview>` でローカル/外部 Web をインライン表示
8. エクスプローラーからファイルタイプに応じた自動ルーティング

### Phase 3: 高度な機能

9. ペインの動的分割・リサイズ
10. コンテキスト共有ブリッジ（ビューア ↔ ターミナル）
11. ワークスペース設定の永続化・復元
12. ブラウザパネルの Playwright 連携（スクリーンショット・DOM操作）

## 6. 未決事項 / 設計上の課題

| 課題 | 検討内容 |
|---|---|
| ブラウザパネルのセキュリティ | `webview` の CSP / CORS制御 |
| PDF のアノテーション | Chromium内蔵以上の機能が必要なら `pdf.js` 導入 |
| ターミナルの永続化 | プロセス再起動時のセッション復元（tmux attach 的な機能） |
